# Checklist: Pilot Launch

> Pre-launch checklist before sharing with real users

---

## Technical Readiness

### Core Functionality

- [ ] Event Node: Can create events
- [ ] Event Node: Can edit events
- [ ] Event Node: Can delete events
- [ ] Event Node: Events sync to Community Node
- [ ] Community Node: Displays events from Event Node
- [ ] Community Node: Can filter events by network
- [ ] Community Node: Can filter events by date
- [ ] Community Node: Users can view event details
- [ ] Community Node: Users can RSVP

### Punch Pass (Demo Mode)

- [ ] Can view available punch passes
- [ ] Can "purchase" punch pass (demo)
- [ ] Pass appears in user account
- [ ] Can use pass for event RSVP
- [ ] Balance updates correctly
- [ ] Clear indication this is "demo mode"

### User Accounts

- [ ] Users can create account
- [ ] Users can log in
- [ ] Users can log out
- [ ] Password reset works (or is disabled)
- [ ] User profile displays correctly

### Error Handling

- [ ] Network errors show friendly message
- [ ] Form validation errors are clear
- [ ] Empty states have helpful messages (e.g., "No events yet")
- [ ] App doesn't crash on unexpected errors

### Mobile

- [ ] All pages load on mobile
- [ ] Navigation works on mobile
- [ ] Forms are usable on mobile
- [ ] Text is readable (not too small)

---

## Content Readiness

### Test Data

- [ ] At least 2-3 test events exist
- [ ] Events have realistic titles/descriptions
- [ ] Events span different dates (upcoming)
- [ ] At least one event is in Recess network
- [ ] At least one event is in Homeschool network

### Punch Passes

- [ ] At least one punch pass type configured
- [ ] Price is realistic (even if demo)
- [ ] Description is clear

---

## User Documentation

### For Event Coordinators

- [ ] How to create an event (written or video)
- [ ] How to edit/delete an event
- [ ] How to enable punch pass acceptance
- [ ] Who to contact for help

### For Community Members

- [ ] How to browse events
- [ ] How to RSVP
- [ ] How punch passes work
- [ ] Who to contact for help

---

## Feedback Collection

- [ ] Way to collect feedback (form, email, or chat)
- [ ] Clear ask: "What's confusing? What's missing?"
- [ ] Plan to review feedback within 48 hours

---

## Rollback Plan

If something goes catastrophically wrong:

- [ ] Know how to unpublish in Base44
- [ ] Know how to revert a GitHub commit
- [ ] Have contact info for pilot users to notify them

---

## Communication

### Pilot User Briefing

- [ ] Explain this is an early pilot
- [ ] Set expectations (things may break)
- [ ] Explain how to report issues
- [ ] Thank them for being early testers

### Talking Points

Be ready to explain:
- [ ] What LocalLane is (30-second pitch)
- [ ] What makes it different (trust, not ads)
- [ ] What the pilot is testing
- [ ] What's coming next

---

## Launch Day

- [ ] All checklist items above complete
- [ ] You've tested the full flow yourself today
- [ ] Pilot users have been briefed
- [ ] You're available to respond to issues

---

## Post-Launch (First 48 Hours)

- [ ] Monitor for critical bugs
- [ ] Respond to pilot user questions quickly
- [ ] Document all feedback
- [ ] Fix critical issues immediately
- [ ] Note non-critical issues for later

---

*Don't skip this checklist. A bad pilot experience is hard to recover from.*
